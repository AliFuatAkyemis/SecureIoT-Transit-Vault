## Project Overview

ESP32-based IoT vault with RFID access control, tamper detection, remote cloud control, and OLED status display. Deployed via Arduino IDE or Arduino Cloud editor.

**Target board:** ESP32 DOIT DevKit V1 (`esp32:esp32:esp32doit-devkit-v1`)

## Build & Upload

Arduino IDE or Arduino Cloud — no CLI build system. Use Arduino IDE 2.x:

1. Open `SecureIoT_Transit_Vault/SecureIoT_Transit_Vault.ino`
2. Select board: **DOIT ESP32 DEVKIT V1**
3. Install required libraries (see below)
4. Fill `arduino_secrets.h` with credentials
5. Upload via USB

To read RFID card UIDs: open Serial Monitor at **115200 baud**, tap a card, copy UID from `[RFID] Card UID:` line, add to `AUTHORIZED_UIDS[]` in `.ino`.

## File Structure

- `SecureIoT_Transit_Vault.ino` — main logic: setup/loop, hardware drivers, RFID auth, motion detection, OLED rendering, cloud callbacks
- `thingProperties.h` — Arduino IoT Cloud auto-generated; defines cloud variables and `initProperties()`. Edit with care — cloud sync can overwrite it.
- `arduino_secrets.h` — secrets: `SECRET_SSID`, `SECRET_OPTIONAL_PASS`, `SECRET_DEVICE_KEY`. Not committed (gitignored).

## Architecture

**Hardware peripherals** (all on I2C bus SDA=21/SCL=22 or SPI):
- OLED SSD1306 @ 0x3C — status display, refreshed every 500ms
- MPU-6050 — accelerometer/gyro; triggers alarm if G > 2.5g or tilt > 45°
- MFRC522 RFID — polled each loop; authorized UIDs hardcoded in `AUTHORIZED_UIDS[]`
- Servo on pin 13 — 0° = locked, 90° = unlocked
- Buzzer on pin 12 — latching alarm (stays ON until cloud `alarmReset`)

**Cloud variables** (Arduino IoT Cloud dashboard):
- `angle`, `gForce`, `lockStatus`, `rfidUID` — READ only (sensor telemetry)
- `remoteUnlock` — READWRITE; rising edge unlocks vault, auto-resets to false
- `alarmReset` — READWRITE; rising edge silences buzzer, auto-resets to false

**Control flow:** `loop()` calls `ArduinoCloud.update()` → `readMotion()` → `readRFID()` → `updateOLED()` (rate-limited). Cloud callbacks `onRemoteUnlockChange()` and `onAlarmResetChange()` fire from within `ArduinoCloud.update()`.

## Key Constraints

- `thingProperties.h` is auto-generated by Arduino Cloud — manual edits may be overwritten. Cloud variable additions must match the dashboard configuration.
- Alarm is **latching**: once triggered by motion, only `alarmReset` via cloud clears it. There is no local reset.
- `AUTHORIZED_UIDS[]` is hardcoded — no runtime enrollment. Add UIDs as uppercase hex strings (e.g., `"A1 B2 C3 D4"`).
